# fb_kr3

## Введение

Проект `fb_kr3` — это PWA-приложение для заметок, реализованное по практическим занятиям №13–18 по дисциплине «Фронтенд и бэкенд разработка».

В проекте реализованы:

- офлайн-работа через Service Worker;
- поддержка установки как PWA через Web App Manifest;
- архитектура App Shell с динамической подгрузкой контента;
- WebSocket-взаимодействие в реальном времени (Socket.IO);
- Push-уведомления и внутренние уведомления внутри браузера;
- напоминания с возможностью отложить уведомление на 5 минут (`/snooze`).

## Инструкция к запуску

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск приложения

```bash
npm start
```

Сервер стартует через `server.js` на порту `3001`.

Приложение открывается по адресу:

- `http://localhost:3001` (или `https://localhost:3001`, если найдены локальные сертификаты).

### 3. Режим разработки (сервер + слежение за SCSS)

```bash
npm run dev
```

### 4. (Опционально) Локальный HTTPS

Если нужны сертификаты для локального HTTPS:

```bash
npm run certificate
```

После генерации сертификатов `server.js` автоматически поднимет `https://localhost:3001` (если файлы сертификатов найдены).

### 5. (Опционально) Ручная генерация Service Worker

```bash
node scripts/generate-sw.js
```

Скрипт находится в проекте и может использоваться для пересборки `sw.js`.

## Описание каждой сделанной практики

### Практика 13 — Service Worker

Реализовано базовое офлайн-приложение заметок:

- регистрация Service Worker;
- кэширование статических ресурсов;
- сохранение заметок в `localStorage`;
- работа приложения без сети (просмотр и добавление заметок).

### Практика 14 — Web App Manifest

Добавлена поддержка установки приложения как PWA:

- настроен `manifest.json` (name, short_name, start_url, display, цвета, description, icons);
- подключен manifest в `index.html`;
- добавлены meta-теги мобильной интеграции;
- иконки включены в стратегию кэширования.

### Практика 15 — HTTPS + App Shell

Реализована архитектура App Shell и подготовка к защищенному запуску:

- каркас приложения в `index.html`;
- динамическая подгрузка `content/home.html` и `content/about.html`;
- стратегия кэширования: статика (Cache First), контент (Network First с fallback);
- поддержка локального HTTPS через сертификаты.

### Практика 16 — WebSocket + Push

Добавлена работа в реальном времени и уведомления:

- сервер на `Express + Socket.IO + web-push`;
- отправка/получение событий при добавлении, изменении и удалении заметок;
- подписка и отписка на push (`/subscribe`, `/unsubscribe`);
- системные push-уведомления через Service Worker;
- внутренние уведомления внутри браузера.

### Практика 17 — Напоминания

Добавлены напоминания для заметок:

- отправка события `newReminder` с временем уведомления;
- серверное планирование напоминаний через `setTimeout`;
- отправка push-уведомления при наступлении времени.

### Практика 18 — Snooze для push-уведомлений

Добавлено отложенное напоминание:

- кнопка действия в push-уведомлении: «Отложить на 5 минут»;
- обработчик `notificationclick` в `sw.js`;
- endpoint `POST /snooze` на сервере для перепланирования напоминания.
